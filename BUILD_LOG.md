## 2026-04-21 — dev-loop fixes: reload-hook system + /restart fully working + logs_scoped

Three fixes in one bounce on top of the pillars wave. (1) **Post-reload hook system** in `nexus_debug_http.py` — modules that monkey-patch another module now call `register_post_reload_hook(module_name, callback)` and the /reload endpoint fires those callbacks after `importlib.reload` succeeds. Response now returns `hooks_fired` and `hooks_errors` arrays for observability. First user: `nexus_lottery.install()` registers its `_install_rare_patch` for `"nexus_mind"`, so `/reload nexus_mind` no longer silently blows away the ~1% gold wrapper. Hook errors are isolated — one bad callback doesn't abort the rest. Verified live: `/reload nexus_mind` returned `hooks_fired: ["_install_rare_patch"]`, log showed the patch re-installed. (2) **/restart now actually works.** Three bugs were stacked. First, the old handler used `threading.Thread(target=time.sleep(0.6) + os._exit(0))` which raced the aiohttp response write — clients saw `null` bodies because os._exit killed the event loop before the JSON reached the socket. Fixed by scheduling exit as an `asyncio.create_task` inside the running loop so the response flushes first. Second, `_spawn_detached_restart` was opening the log file directly from the parent Python (`open(log_path, "ab")`), which hit `PermissionError [Errno 13]` on Windows because the parent's cmd.exe wrapper already had a share-exclusive handle on the same file. Fixed by using `subprocess.Popen(["powershell.exe", "-Command", "Start-Process … -WindowStyle Hidden"])` so the grandchild cmd.exe owns its own redirect — same pattern as `nx start`. Third, even with the new spawn, the grandchild's cmd would race the parent to open the log (parent still had it for ~1s until exit fired) and silently lose. Fixed by prefixing the grandchild's command with `timeout /t 2 /nobreak > nul &&` so it waits 2s before attempting the redirect, guaranteeing the parent has died and released the log. End-to-end verified: `POST /restart` → proper JSON body with old/child pids → new process alive 2s later → old process gone → /ping returns the new pid after boot. `/kill` got the same asyncio-task exit treatment for consistency. (3) **`POST /logs_scoped`** added to `nexus_logs_catchup.py` — takes `{user_ids: [...]}`, resets `#📝│logs` overwrites back to Architect-only default, then adds per-user view overrides for the named members. Used to replace the earlier broad `/logs_void` flip with a scoped grant: a member (id `[redacted]`) can now read `#📝│logs` as a user-level exception without getting promoted to Architect (which would also give him Void→Signal promotion power). Other Void/Signal members lose access. Channel perms applied cleanly on first try.
